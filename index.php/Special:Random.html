Installing Amber14 on the group cluster[edit]

While the instructions below specifically describe compiling amber14 on zin using the Intel 12 compiler, this is also a good start if you're trying to perform a local installs or work with available Intel compiler libraries. First off, gcc (and g++/gfortran) 4.7.4 is currently installed and is run as the default c, c++, and fortran compiler. However, amber will purportedly run faster when compiled with an Intel compiler, the process for which is outlined here.

1) First step is to obtain the AmberTools15 and Amber14 packages, unpacking with 

     tar xvfj Amber14.tar.bz2

The code from both packages should be placed in a directory called amber14.

2) The available Intel 12.0.4 compiler and its associated libraries are not on the default path. The path may be appropriately updated with the following code: 

   
   export AMBERHOME=$PWD
   export CPLUS_INCLUDE_PATH=/home/cask0/home/jmonroe/bin/gcc4.6/include/c++/4.6.3:$LD_LIBRARY_PATH
   export LD_LIBRARY_PATH=/share/apps/x86_64/intel/composerxe-2011.4.191/lib/intel64:/home/cask0/home/jmonroe/bin/gcc4.6/lib64:$LD_LIBRARY_PATH
   export PATH=/share/apps/x86_64/intel/composerxe-2011.4.191/bin:/home/cask0/home/jmonroe/bin/gcc4.6/bin:$PATH
   export MKL_HOME=/share/apps/x86_64/intel/composerxe-2011.4.191/mkl
   source /share/apps/x86_64/intel/composerxe-2011.4.191/bin/iccvars.sh intel64

This may be placed in a script and run with source above_script.sh or directly from the command line. The first line assumes the user is in the amber14 directory, otherwise replace $PWD with /path/to/amber14/. The next line ensures that the correct c++ libraries linked to the correct version of gcc are used (more on this below). The LD_LIBRARY_PATHvariable defines the default libraries that are read. The Intel 12.0.4 libraries and correct gcc version library directories must be added to this. PATH is adjusted to include the Intel compilers for c, c++, and fortran. The last couple of lines ensure that the Math Kernel libraries and correct settings associated with the most up to date Intel compiler that we have (version 12.0.4) are used.

3) One problem: Intel 12.0.4 is not compatible with gcc 4.7.4 libraries that must be accessed. So, an older version must be used. Below is how to do a quick local install of gcc 4.6.3. Once the package is obtained (try, for example, gcc-4.6.3.tar.gz from [1]) and unpacked, likely to gcc-4.6.3, create a target directory where you want the code to be compiled to. This should be owned by you and off your home directory somewhere. Also create a build-gcc directory to use while compiling. A good place for this is on the same level as the directory which contains the unpacked source code. Once in this build directory, try 

    ./configure --prefix=/path/to/target/compile/local/dir

If any packages such as GMP, MPFR, or MPC are missing, a trick to quickly get them for only this local install is to go into the unpacked gcc-4.6.3 directory and run 

    ./contrib/download_prerequisites

This should do the trick, and you should hopefully be able to run the previous configure script without issues. Then run 

   
   make
   make install

Note that this will take a long time, but at the end, you should have a local install of gcc 4.6.3. Also, please not that most any program can be compiled and installed locally with the --prefix= configure option, though you should consult the website of the specific software to make sure.

4) Compiling and installing amber14, without mpi, is now easy. Just go to your amber14 directory and type 

   
   ./configure intel
   make install

If you want to compile with mpi (multiple processor) capabilities, keep going...

5) Compiling with mpi will require downloading openmpi and placing it the appropriate directory within amber14. Instructions are in the amber14 manual, but you can just got to /path/to/amber14/AmberTools/src and type 

    ./configure_openmpi intel

Then follow the instructions presented there. After this, make sure that your LD_LIBRARY_PATH variable is updated to include /path/to/amber14/lib. It will likely be necessary to also add /path/to/amber14/bin to PATH. Then just run 

   
   ./configure -mpi intel
   make install

6) All done! To use amber14 anytime, type the following into the command line or include in a job submission script to correctly set-up your environment for its use: 

   
   export AMBERHOME=/home/cask0/home/jmonroe/bin/amber14 #or wherever it's just been installed
   export PATH=$AMBERHOME/bin:$PATH
   export SANDER=$AMBERHOME/bin/sander
   export PMEMD=$AMBERHOME/bin/pmemd
   export LD_LIBRARY_PATH=$AMBERHOME/lib:$LD_LIBRARY_PATH
   export MPI_HOME=$AMBERHOME
   export MKL_HOME=/share/apps/x86_64/intel/composerxe-2011.4.191/mkl
   source /share/apps/x86_64/intel/composerxe-2011.4.191/bin/iccvars.sh intel64
